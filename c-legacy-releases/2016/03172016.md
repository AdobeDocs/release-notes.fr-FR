# Notes de mise à jour - mars 2016

Découvrez les nouvelles fonctionnalités et les correctifs d’Adobe Marketing Cloud.

>[!NOTE]
>
>pour recevoir les notes de mise à jour une semaine avant la mise à jour mensuelle, abonnez-vous à [Adobe Priority Product Update](https://www.adobe.com/subscription/priority-product-update.html), où les informations de mise à jour sont publiées une semaine avant la date de publication et peuvent être modifiées. Revenez-y au moment de la publication de la mise à jour pour connaître les dernières informations.

## Experience Cloud

Notes de mise à jour concernant Experience Cloud et les services principaux.

### Adobe Mobile Services (9 mars 2016)   {#mobile}

Le kit SDK iOS version 4.8.6 (du 9 mars 2016) contient les modifications suivantes :

| Fonction | Description |
|--- |--- |
| Suivi des plantages d’application | La version 4.8.6 du SDK iOS comporte des modifications critiques qui empêchent le signalement de faux plantages. Nous vous recommandons vivement d’installer la version 4.8.6. |

Pour consulter la documentation du produit, reportez-vous à la section [Adobe Mobile Services](https://marketing.adobe.com/resources/help/en_US/mobile/). Pour afficher les notes de mise à jour des versions précédentes, développez *Notes de mise à jour précédentes* dans le volet gauche.

### Service Marketing Cloud ID {#section_5BA06A92D6E54715B025D86C192B2A8B}

17 mars 2016

La version 1.5.4 comprend les modifications suivantes :

| Fonction | Description |
|--- |--- |
| Prise en charge de l’exclusion | Le service Marketing Cloud ID prend en charge les demandes d’exclusion des visiteurs. (AAM-21725) |
| Modification de l’intervalle de synchronisation d’ID | Le service Marketing Cloud ID effectue désormais des appels de synchronisation d’ID à chaque appel aux serveurs de collecte de données. Le service d’ID n’effectuait auparavant qu’une seule demande lors du premier appel pour obtenir un Marketing Cloud ID. (AAM-22934) |

**Correctifs, améliorations et obsolescences**

* Correction d’un bogue en raison duquel les rapports de tendance présentaient des résultats décalés d’un jour par rapport à la date en cours. Concerne uniquement les fuseaux horaires de l’Europe de l’Est. (AAM-22321)
* Correction d’un bogue dans Audience Marketplace qui empêchait les fournisseurs de données d’activer un flux de données privé sans marque. (AAM-23377)
* L’option de type d’ID dans le processus de création de source de données à été remplacée par Device Advertising ID (Identifiant de publicité de périphérique) (auparavant : Mobile). (AAM-22947)
* Ajout d’un nouvel opérateur nommé ` mathcesword` qui aide à créer des règles de caractéristique sans la complexité des expressions régulières. (AAM-22368)
