## [!DNL Audience Manager]

**Correctifs, améliorations et obsolescences**

* Améliorations des [!UICONTROL règles de fusion de profils] : tous les appareils admissibles pour un segment en temps réel qui utilisent une règle de fusion de profils créée avec une option de graphique de périphérique (p. ex. le graphique d’Adobe Co-op Device ou un graphique de périphérique externe) seront désormais inclus dans le décompte Population totale du segment. Par ailleurs, les identifiants de ces périphériques qualifiés seront également envoyés à toutes les destinations par lots configurées pour ce segment. Voir [FAQ relative aux règles de fusion de profils et les graphiques de périphérique](https://marketing.adobe.com/resources/help/en_US/aam/profile-merge-faq.html) et [Données de population des segments et caractéristiques dans le Créateur de segments](https://marketing.adobe.com/resources/help/en_US/aam/segment-builder-data.html). (AAM-27726)
* Placement des données de chaîne entre guillemets doubles dans les fichiers CSV pour les [!UICONTROL rapports de chevauchement]. Voir [Fichiers CSV pour les rapports de chevauchement](https://marketing.adobe.com/resources/help/en_US/aam/?f=overlap-csv-files.html).

**Correctifs de bogues DIL, améliorations et code obsolète**

Le code DIL constitue le code de collecte de données d’API d’Audience Manager. La mise à jour de juillet incrémente le code DIL vers la version 6.10. Cette version inclut un code révisé permettant de réduire l’utilisation du processeur grâce à des améliorations de l’efficacité des appels à l’iFrame de publication de destination. (MCID-419)
