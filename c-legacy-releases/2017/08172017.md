**Correctifs**

* Corrections de bugs mineurs et améliorations des performances
Pour obtenir de la documentation sur le produit, voir [Services d’Adobe Mobile](https://marketing.adobe.com/resources/help/en_US/mobile/) . Pour afficher les notes de mise à jour des versions précédentes, développez *Notes de mise à jour précédentes* dans le volet gauche.

**Mise à jour 23 août 2017**

Depuis le 22 août, Adobe Analytics définit l’attribut geo_zip sur tous les accès reçus durant la collecte des données. Les données sont ainsi plus précises, en particulier celles qui sont issues des appareils mobiles. Auparavant, l’attribut geo_zip était défini une seule fois par visite. Il peut désormais changer durant la visite d’un visiteur. L’attribut geo_zip n’est pas utilisé dans les rapports, mais apparaît dans les flux de données.

**Mise à jour 21 août 2017**

Le 22 août, Adobe Analytics activera les rapports de [géo-segmentation](https://marketing.adobe.com/resources/help/en_US/reference/reports_geosegmentation.html) pour tous les clients. Ce changement intervient afin d’anticiper la suppression prochaine des paramètres de géosegmentation dans Admin Console.

Date de publication : **17 août 2017**

>[!IMPORTANT]
>
>À partir de septembre 2017, Ad Hoc Analysis ne prendra plus en charge les versions Java antérieures à 1.7.0_76. Nos fichiers .jar sont protégés par un cryptage sécurisé de 256 bits qui n’est pas pris en charge par les versions de Java antérieures à 1.7.0_76. Ce certificat 256 bits nous permet de vous offrir une sécurité optimale. La version d’août 2017 continuera à prendre en charge les versions antérieures. En outre, à partir de septembre 2017, la page de lancement d’Ad Hoc Analysis inclura temporairement un lien vers une version non prise en charge du produit, à utiliser avec les versions de Java antérieures à 1.7.0_76. Si vous utilisez Java 1.7.0_76 ou une version antérieure, veuillez demander à votre équipe informatique d’effectuer une mise à niveau vers une version plus récente de Java, dans les plus brefs délais.

**Correctifs, améliorations et obsolescences**

* Correction d’un bogue visuel dans la vue de destination, en raison duquel une icône d’info-bulle supplémentaire apparaissait chaque fois qu’un utilisateur sélectionnait une autre valeur dans la fenêtre déroulante Périphériques. (AAM-28010)
* Les utilisateurs liés avec des [identifiants d’entreprise](https://marketing.adobe.com/resources/help/en_US/aam/ids-in-aam.html) peuvent désormais se connecter à l’interface utilisateur d’Audience Manager à l’aide de leur Adobe ID plutôt que de leur nom d’utilisateur et de leur mot de passe. (AAM-28161)
* L’écran de recherche Audience Marketplace comprend désormais un lien vers [Audience Finder](https://www.adobe-audience-finder.com/). Grâce à Audience Finder, trouvez des partenaires susceptibles de vous fournir le bon type de flux de données pour votre campagne. (AAM-30241)
* Correction d’un problème en raison duquel les utilisateurs pouvaient supprimer les mappages de segments de test dans Audience Lab, même s’ils étaient utilisés dans une destination. (AAM-29964)
* Les utilisateurs administrateurs reçoivent un message électronique automatisé quand l’accès à un flux de données partagées est révoqué. Les messages électroniques comprennent maintenant un lien vers le site de documentation du produit, qui explique pourquoi l’accès a été révoqué. (AAM-30625)
* Correction d’un bogue en raison duquel la fenêtre modale des destinations ne s’agrandissait pas au-delà de la taille de la fenêtre du navigateur. (AAM-30893)
* Les caractéristiques du dossier sont exclues de la liste des caractéristiques d’influence dans la modélisation algorithmique. (AAM-30960)
* Correction d’un bogue en raison duquel la suppression d’un mappage de destination prenait beaucoup de temps. (AAM-31338)
* Correction d’une erreur d’interface utilisateur en raison de laquelle les mesures connexes d’audience adressables affichent zéro au lieu de N/A pour les mesures non applicables. (AAM-31712)
* Correction d’un problème en raison duquel l’interface utilisateur était longue à répondre pour les pages multiples dans le cas de clients ayant des centaines de sources de données configurées. (AAM-31788)

* Nous avons ajouté une nouvelle configuration ` isCoopSafe`. Il s’agit d’une configuration booléenne facultative permettant de déterminer si DIL envoie des données à [!DNL  Adobe Marketing Cloud Device Co-op]. Voir [isCoopSafe](https://marketing.adobe.com/resources/help/en_US/aam/?f=dil-coopsafe.html). (CORE-4334)
